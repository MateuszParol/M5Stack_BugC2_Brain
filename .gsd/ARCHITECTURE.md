## Overview

BugC2 Brain is a C++ firmware for the M5StickC Plus 2 and an AtomS3R-CAM vision node that controls a BugC2 robotic base. The system allows an operator to steer the robot via a Web UI (WiFi AP mode) while safety subsystems running on the main MCU ensure collision avoidance using a Time-of-Flight sensor. An experimental AI camera module running separate firmware detects obstacles and feeds data via I2C to the main brain. It also features a rudimentary autonomous exploration mode.

## Data Flow

1. **Hardware Initialization**: The system starts `M5`, `Wire`, `MotorDriver`, `ToFSensor`, `VisionSensor`, and `RemoteController`. UI is initialized.
2. **Sensor Polling**: Within the `loop()`, the `ToFSensor` and `VisionSensor` update their internal states.
3. **Collision Detection**: The `CollisionDetector` evaluates distance flags and classifies them into safe, warning, or critical zones.
4. **Command Reception**: Either `RemoteController` processes an HTTP request for movement, or `AutoNavigator` generates an autonomous movement vector.
5. **Actuation & Override**: Commands are evaluated against `CollisionDetector` outputs. If a collision is imminent, forward moves are blocked or overridden with stops/reverses. Valid commands are written over I2C to the BugC2 motor controller.
6. **Telemetry Updates**: LCD state (`DisplayManager`) and LED indications are refreshed at specified intervals.
